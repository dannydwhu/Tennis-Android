# Smartform 网球训练 App - 产品需求与 UI 设计规范文档

## 第一部分：品牌定位与设计愿景

### 1.1 产品定位
- **产品名称**：Smartform（智塑科技 · 网球运动传感器配套 App）
- **品牌调性**：专业、科技、精准、活力
- **核心价值**：Smartform 不是普通的运动记录工具，而是一个网球运动员的**专业训练伙伴**。界面应传达出专业运动科技感——数据精准、视觉清晰、交互流畅，让用户在训练中感受到「被专业对待」的体验。
- **设计语言关键词**：精准数据可视化、运动动感、简洁不简陋、深色为主降低干扰。

### 1.2 设计原则
1. **数据优先**：击球个数、速度等核心数据必须醒目、实时更新。
2. **沉浸体验**：深色背景降低户外/夜训时的屏幕干扰。
3. **专业反馈**：动作分类、能力评估、热力图等专业数据用图表清晰表达。
4. **动感交互**：击球瞬间有微动效反馈，速度峰值有高亮动画。

## 第二部分：UI 设计规范体系

### 2.1 色彩体系

| 用途 | 色值 | 说明 |

## 第三部分：功能模块详细拆分

基于 TennisACE 设计图，我们将 App 划分为五大核心页面，每个页面进一步细化组件与交互逻辑。

### 3.1 底部导航栏（全局）
- **Tab 项**：LIVE（实况）、数据、分析、排行榜、我的
- **样式**：深色背景 `#0D1117`，选中态品牌绿 `#00D68F`，未选中态辅助文字 `#FFFFFF66`
- **高度**：48px + 安全区域

---

### 3.2 实况页（LIVE）—— 训练核心页面

| 组件 | 功能描述 | UI 规范细节 | 交互逻辑 |
| :--- | :--- | :--- | :--- |
| **开始训练按钮** | 连接蓝牙硬件，进入训练状态 | 品牌绿渐变填充，圆角 12px，高度 48px，文字白色粗体 | 点击后检测蓝牙连接，未连接则引导连接；连接成功后进入训练界面 |
| **六项技术动作卡片** | 实时显示正手、反手、切削、高压/发球、正手截击、反手截击的累计个数 | 网格布局 2x3，卡片背景 `#161B22`，圆角 12px，内边距 12px。图标使用对应语义色，个数使用 28px Semibold | 每次算法检测到击球，对应卡片数字 +1，并伴随微弹跳动动画 |
| **最大挥速仪表盘** | 显示当前或最近一次击球的最大速度（km/h） | 大号数字 64px Bold，单位 16px，居中显示。速度峰值时数字短暂变为 `#FF1744` 并放大 | 实时更新，速度超过用户历史平均时触发成就反馈 |
| **连击计数器** | 显示连续有效击球次数 | 小卡片形式，位于速度仪表盘下方，连击中断时归零 | 两次击球间隔 <2s 计为连击，超过则清零并记录最高连击 |
| **训练时长 / 卡路里** | 计时器和估算卡路里消耗 | 辅助文字 14px，次要文字色 `#FFFFFF99`，位于顶部状态栏下方 | 训练开始时计时，卡路里基于体重和动作频率估算 |
| **心率（预留）** | 显示实时心率（如连接心率带/手表） | 小图标 + 数字，初期可隐藏或显示 "--" | V1.1 对接 HealthKit / Google Fit |
| **结束训练** | 终止本次训练并生成报告 | 按钮置于底部，红色 `#FF1744`，圆角 12px | 点击后结束会话，自动跳转至本次训练报告 |

### 3.3 数据页（DATA）—— 历史统计与能力评估

| 组件 | 功能描述 | UI 规范细节 | 交互逻辑 |
| :--- | :--- | :--- | :--- |
| **时间切换 Tab** | 今日、本周、本月、本年 | 顶部切换栏，选中态品牌绿下划线，文字 Semibold | 切换后刷新下方所有数据卡片 |
| **核心指标卡片** | 击球数、卡路里、运动时长、最大挥速等 | 采用数据卡片网格，背景 `#161B22`，圆角 20px。主数据 28px Bold，标签 14px 辅助色 | 静态数据展示，支持下拉刷新同步最新数据 |
| **能力分析五维图** | 爆发力、进攻性、活跃度、对抗性、耐力 | 雷达图 / 五边形图，填充品牌绿 `#00D68F` 半透明，边框实色。各维度 0-10 分 | 算法根据用户历史训练数据计算，V1.0 使用规则引擎计算伪数值 |
| **挥速区间分布** | 展示用户挥速的直方图分布 | 柱状图，柱体使用品牌蓝 `#3366FF`，带渐变效果 | 帮助用户了解自己挥速的集中区间 |
| **目标设定** | 设定运动时长、消耗卡路里目标 | 卡片内嵌进度条，品牌绿填充，显示完成百分比 | 点击进入目标设定弹窗，保存后同步至我的页面 |

### 3.4 分析页（ANALYSIS）—— 视频动作分析

| 组件 | 功能描述 | UI 规范细节 | 交互逻辑 |
| :--- | :--- | :--- | :--- |
| **可用次数展示** | 显示剩余视频分析次数 | 顶部卡片，品牌蓝背景，数字 64px Bold | 消耗型功能，可通过分享或购买获得次数 |
| **分析类型入口** | 发球分析、正手分析、反手分析 | 卡片列表，每个卡片带图标、标题、说明文字，右箭头 | 点击进入视频上传流程 |
| **视频上传** | 录制或选择相册中 4 秒内视频 | 系统原生选择器，上传进度条 | 上传至 OSS，后台 AI 分析后返回分数和建议 |
| **分析结果页** | 展示动作分数、详细建议、关键帧标注 | 评分用大号数字，建议用列表形式，关键帧可左右滑动 | V1.0 仅支持发球/正手/反手，后续扩展更多动作 |

## 第四部分：开发周期调整

基于上述细化的 UI 与功能模块，结合之前 17 周计划，进行微调：

| 阶段 | 周次 | 核心任务 | 关键交付物 |
| :--- | :--- | :--- | :--- |
| **Phase 1: 基础框架** | W1-W3 | 项目初始化、设计规范落地（颜色/字体/间距组件封装）、底部导航搭建、蓝牙 SDK 集成 | 具备设计规范的基础 UI 框架，蓝牙连接 Demo |
| **Phase 2: 实况页闭环** | W4-W6 | 实况页 UI 开发、算法数据接入、六项卡片实时刷新、速度仪表盘、训练会话管理 | 实况页完整功能，能与算法联调跑通训练流程 |
| **Phase 3: 数据与报告** | W7-W9 | 数据页图表开发（五维图、直方图）、训练报告生成与分享、历史记录列表 | 数据页完整，可查看历史报告并分享 |
| **Phase 4: 用户体系与排行榜** | W10-W12 | 登录注册（手机/微信）、首次资料填写、成长系统、排行榜 UI 与接口联调 | 用户可登录，排行榜可用 |

| **Phase 6: 联调与上架** | W15-W17 | 全流程测试、Bug 修复、性能优化、商店素材准备、提交审核 | 提交 App Store 和主流安卓市场 |

## 第五部分：设计规范交付物清单

为确保开发与设计一致，UI 设计侧需提供以下资源：

1. **全局样式文件**：颜色变量表（XML / JSON）、字体样式表（TextStyle）、尺寸常量表（dimens.xml）
2. **组件库**：按钮、卡片、进度条、标签、图表等可复用组件的设计稿与代码示例
3. **图标资源**：六项击球动作图标、底部导航图标（选中/未选中）、通用功能图标，提供 SVG / PNG @2x @3x
4. **关键页面设计稿**：实况页、数据页、分析页、排行榜、我的、训练报告，标注好间距与字体
5. **动效说明**：击球计数动画、速度峰值变色、页面转场效果描述
